| Attack Surface                  | Description                                                                                                          | Techniques Used                                                                                                                               | Command(s)                                                                                                          | How to Test                                                                                                                                           |
|---------------------------------|----------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| **EAPOL-Start Flooding**         | Overwhelms the authentication server with a flood of EAPOL-Start messages, potentially causing denial of service    | EAPOL Start Flooding                                                                                                                          | `mdk4 mon0 e -t <MAC_AP>`                                                                                           | Simulate the attack against a test environment and monitor the authentication server for performance impact                                           |
| **EAPOL-Logoff Flooding**        | Sends a flood of EAPOL-Logoff messages to disconnect clients                                                         | EAPOL Logoff Injection                                                                                                                         | `mdk4 mon0 e -t <MAC_AP> -l`                                                                                        | Simulate the attack and observe if connected clients get disconnected                                                                                |
| **Deauthentication/Disassociation Attack** | Forces clients to reauthenticate, disrupting connectivity                                                      | Deauthentication / Disassociation                                                                                                              | `aireplay-ng --deauth 20 -c <MAC_target> -a <MAC_AP> mon0` <br> `mdk4 mon0 d -c <channel> -E <SSID>`                | Simulate the attack and observe if connected clients get disconnected and have to reauthenticate                                                     |
| **Rogue AP Attack**              | Sets up a rogue AP to intercept client connections and authentication requests                                      | Rogue Access Point                                                                                                                              | `airbase-ng -e <SSID> -c <channel> mon0`                                                                            | Set up a rogue AP and observe if clients attempt to connect to it instead of the legitimate AP                                                      |

| **Downgrade Attack**             | Forces the client to use a less secure version of the authentication protocol                                        | Protocol Downgrade                                                                                                                               | Modify AP settings or use a rogue AP to force the client to connect with a less secure protocol version             | Attempt to force a client to use a less secure protocol version and observe if it succeeds                                                          |
| **EAP Relay Attack**             | Relays EAP messages between client and server without altering them, often to bypass client-side security checks     | EAP Relay                                                                                                                                        | Use tools like `ettercap`, `mitmproxy` configured for EAP relay                                                    | Set up an EAP relay attack and observe if you can bypass client-side security checks                                                                |
| **LEAP Cracking**                | Exploits weaknesses in LEAP to recover user credentials                                                             | LEAP Dictionary Attack                                                                                                                           | Use `asleap` tool for LEAP cracking                                                                                 | Capture LEAP traffic and attempt to crack it using the `asleap` tool                                                                                |
| **PEAP/EAP-TTLS Cracking**       | Exploits vulnerabilities in PEAP or EAP-TTLS to capture and crack user credentials                                  | PEAP/EAP-TTLS Cracking                                                                                                                           | Capture PEAP/EAP-TTLS traffic and use tools like `hashcat` or `asleap` to crack                                     | Capture PEAP/EAP-TTLS traffic and attempt to crack it using tools like `hashcat` or `asleap`                                                        |
| **Fake AP/SSID Spoofing**        | Creates a fake AP with the same SSID as a legitimate one to trick clients into connecting                           | Fake AP Creation                                                                                                                                | `airbase-ng -e <SSID> -c <channel> mon0`                                                                            | Set up a fake AP with the same SSID as a legitimate one and observe if clients attempt to connect to it                                           |
| **EAP Identity Theft**           | Uses a stolen or spoofed identity to gain access to the network                                                     | EAP Identity Theft                                                                                                                               | Modify identity in supplicant configuration or use tools like `wpa_supplicant` with a stolen identity               | Attempt to connect to the network using a stolen identity and observe if it succeeds                                                               |
| **Session Hijacking**            | Takes over an authenticated session by capturing and using session-specific keys                                    | Session Key Hijacking                                                                                                                            | Capture session keys and use tools like `aircrack-ng`                                                               | Capture session keys and attempt to hijack an authenticated session using tools like `aircrack-ng`                                                  |
| **Key Reinstallation Attack (KRACK)** | Exploits weaknesses in the WPA2 protocol to reinstall keys, potentially decrypting traffic                       | KRACK Attack                                                                                                                                     | Use tools like `krackattacks-scripts` to perform the attack                                                         | Use the `krackattacks-scripts` tool to attempt the KRACK attack and observe if it succeeds in decrypting traffic                                     |
| **EAP-Failure Flooding**         | Sends a flood of EAP-Failure messages to disrupt client connections                                                 | EAP-Failure Injection                                                                                                                            | Use custom scripts or tools to send EAP-Failure messages                                                            | Flood the network with EAP-Failure messages and observe if it disrupts client connections                                                          |
| **EAPOL-Key Replay Attack**      | Replays captured EAPOL-Key frames to disrupt or hijack sessions                                                     | EAPOL-Key Replay                                                                                                                                 | Use tools like `aircrack-ng` to capture and replay EAPOL-Key frames                                                 | Capture EAPOL-Key frames and attempt to replay them to disrupt or hijack sessions                                                                   |
| **Beacon Flooding**              | Sends a flood of beacon frames to create fake APs, causing confusion and disrupting network services                | Beacon Flooding                                                                                                                                  | `mdk4 mon0 b -a -w nta`                                                                                            | Flood the network with beacon frames and observe if it creates fake APs and disrupts services                                                      |
| **Authentication Request Flooding** | Floods the AP with authentication requests to overwhelm and disrupt service                                      | Authentication Flooding                                                                                                                          | `mdk4 mon0 a -m -a <MAC_AP>`                                                                                        | Flood the AP with authentication requests and observe if it overwhelms and disrupts the service                                                    |
| **RADIUS Server Attack**         | Targets the RADIUS server to disrupt authentication or obtain credentials                                          | Password Cracking, Denial of Service, Credential Theft                                                                                        | Use tools like `radpenguin` or `radmin` to attack the RADIUS server                                                | Set up a test RADIUS server and attempt various attacks against it                                                                                  |
| **Evil Twin AP**                 | Creates a rogue AP that mimics a legitimate one, tricking clients into connecting                                  | Evil Twin AP Creation                                                                                                                          | Use tools like `airbase-ng` or `hostapd` to create an evil twin AP                                                | Set up an evil twin AP and observe if clients attempt to connect to it                                                                             |
| **PMKID Attack**                 | Captures and cracks the PMKID to recover the PSK for a WPA/WPA2 network                                            | PMKID Capture and Cracking                                                                                                                     | Use `hcxdumptool` and `hashcat` to capture and crack the PMKID                                                     | Capture the PMKID and attempt to crack it using `hashcat`                                                                                          |
| **WPS Brute Force Attack**       | Exploits weaknesses in WPS to recover the network's PSK                                                            | WPS Brute Force                                                                                                                                 | Use tools like `reaver` or `bully` to brute force the WPS PIN                                                     | Attempt a WPS brute force attack against a test AP with WPS enabled                                                                                |
| **Karma Attack**                 | Creates a fake AP that masquerades as previously connected networks to capture credentials                         | Karma Attack                                                                                                                                    | Use `karma` tool or `airbase-ng` with the `--karma` option                                                         | Set up a Karma attack and observe if clients attempt to connect and provide credentials                                                            |
| **EAPoL Packet Analysis**        | Analyzes EAPoL packets to identify weaknesses or obtain information                                                | EAPoL Packet Analysis                                                                                                                           | Use tools like `Wireshark` or `tcpdump` to capture and analyze EAPoL packets                                      | Capture EAPoL packets and analyze them for potential weaknesses or information leaks                                                                |
| **802.1X Brute Force Attack**    | Attempts to guess valid credentials by trying multiple username/password combinations                              | Brute Force Credential Guessing                                                                                                                 | Use tools like `THC-Hydra` or custom scripts to brute force credentials                                            | Attempt a brute force attack against a test 802.1X environment with known credentials                                                             |
| **Rogue RADIUS Server**          | Sets up a rogue RADIUS server to capture or manipulate authentication requests                                     | Rogue RADIUS Server                                                                                                                             | Use tools like `FreeRADIUS` or `radsecproxy` to set up a rogue RADIUS server                                     | Set up a rogue RADIUS server and observe if it can intercept or manipulate authentication requests                                                |

These additional attack vectors cover various techniques, such as targeting the RADIUS server, creating evil twin APs, cracking WPA/WPA2 keys, exploiting WPS vulnerabilities, performing Karma attacks, analyzing EAPoL packets, brute-forcing credentials, and setting up rogue RADIUS servers.

Please note that testing or simulating these attacks should only be performed in a controlled and authorized environment, as they may involve potentially disruptive or illegal activities if conducted without proper permission.
